Memahami Application Programming Interface (API)

API adalah mekanisme yang memungkinkan dua komponen perangkat lunak untuk saling berkomunikasi menggunakan serangkaian definisi dan protokol.

Apa itu API?

Definisi API

API (Application Programming Interface) adalah mekanisme yang memungkinkan dua komponen perangkat lunak untuk saling berkomunikasi menggunakan serangkaian definisi dan protokol.

Misalnya, sistem perangkat lunak badan meteorologi, klimatologi, dan geofisika (BMKG) berisi data cuaca harian. Aplikasi cuaca di ponsel Anda "berkomunikasi" dengan sistem ini melalui API dan menampilkan pembaruan cuaca harian di ponsel Anda.

Kepanjangan API

API merupakan kepanjangan dari Application Programming Interface (Antarmuka Pemrograman Aplikasi).

Kata "Aplikasi" pada API merujuk pada perangkat lunak dengan fungsi yang berbeda. Kata "Antarmuka" dapat diartikan sebagai kontrak layanan antara dua aplikasi. Kontrak ini menjelaskan cara keduanya saling berkomunikasi dengan menggunakan permintaan (request) dan respons (response).

Komponen Utama API

API terdiri dari beberapa komponen utama yang membentuk fungsionalitasnya:

  • Endpoint: URL yang digunakan untuk mengakses API
  • Metode HTTP: GET, POST, PUT, DELETE, dll.
  • Header: Informasi tambahan seperti autentikasi
  • Body: Data yang dikirimkan dalam permintaan
  • Response: Data yang dikembalikan oleh API
  • Status Code: Kode yang menunjukkan status permintaan (200, 404, 500, dll.)

Bagaimana Cara Kerja API?

Arsitektur API

Arsitektur API biasanya dijelaskan dalam kaitannya dengan klien dan server. Aplikasi yang mengirimkan permintaan disebut sebagai klien dan aplikasi yang mengirimkan respons disebut sebagai server.

Perkembangan API

API SOAP

API ini menggunakan Simple Object Access Protocol. Klien dan server saling bertukar pesan menggunakan XML. API yang kurang fleksibel ini populer di masa lalu.

Karakteristik:

  • Menggunakan XML untuk pertukaran data
  • Berbasis protokol (biasanya HTTP atau SMTP)
  • Mendefinisikan struktur pesan yang ketat
  • Mendukung keamanan WS-Security

API RPC

API ini disebut sebagai Panggilan Prosedur Jarak Jauh (Remote Procedure Calls). Klien menjalankan fungsi (atau prosedur) pada server, dan server akan mengirimkan output kembali ke klien.

Jenis:

  • XML-RPC: Menggunakan XML untuk mengodekan panggilan
  • JSON-RPC: Menggunakan JSON untuk mengodekan panggilan
  • gRPC: Dikembangkan oleh Google, menggunakan Protobuf

API Websocket

API WebSocket mendukung komunikasi dua arah antara aplikasi klien dan server. Server dapat mengirimkan pesan callback ke klien yang terhubung, menjadikannya lebih efisien daripada API REST.

Keunggulan:

  • Komunikasi real-time
  • Overhead rendah
  • Persisten koneksi
  • Ideal untuk aplikasi chat, notifikasi real-time

API REST

API REST merupakan API yang paling populer dan fleksibel di web saat ini. Klien akan mengirimkan permintaan ke server sebagai data. Server akan menggunakan input klien untuk memulai fungsi internal dan mengembalikan data output ke klien.

Karakteristik:

  • Stateless
  • Menggunakan metode HTTP standar
  • Resource-based
  • Representasi data fleksibel (JSON, XML)

Jenis-Jenis API

Klasifikasi Berdasarkan Aksesibilitas

Public API

API yang terbuka untuk umum dan dapat diakses oleh siapa saja tanpa batasan signifikan

Private API

API yang digunakan secara internal dalam sebuah organisasi untuk meningkatkan kolaborasi

Partner API

API yang hanya tersedia untuk mitra bisnis tertentu melalui perjanjian khusus

Composite API

Kombinasi dari berbagai API yang memungkinkan pengembang mengakses beberapa endpoint dalam satu panggilan

API Berdasarkan Tingkat Keamanan

Open API

Tanpa autentikasi atau dengan autentikasi minimal

Authenticated API

Membutuhkan autentikasi pengguna untuk mengakses

Secured API

Menggunakan enkripsi dan protokol keamanan tingkat tinggi

API REST - Yang Paling Populer

Apa itu API REST?

REST merupakan kepanjangan dari Representational State Transfer. REST mendefinisikan fungsi-fungsi seperti GET, PUT, DELETE, dll. yang dapat digunakan klien untuk mengakses data server. Klien dan server saling bertukar data dengan menggunakan HTTP.

Fitur utama API REST adalah sifat stateless-nya. Bersifat stateless berarti server tidak menyimpan data klien di antara permintaan. Permintaan klien ke server mirip dengan URL yang Anda ketik di peramban untuk mengunjungi sebuah situs web.

Apa itu API Web?

API Web atau API Layanan Web merupakan sebuah antarmuka pemrosesan aplikasi antara server web dan peramban web. Semua layanan web merupakan API, tetapi tidak semua API merupakan layanan web. API REST merupakan tipe khusus API Web yang menggunakan gaya arsitektur standar.

Contoh penggunaan:

  • Integrasi dengan aplikasi pihak ketiga
  • Membangun aplikasi mobile backend
  • Mikroservis arsitektur
  • Single Page Applications (SPA)

Keuntungan REST API

Integrasi

API digunakan untuk mengintegrasi aplikasi baru dengan sistem perangkat lunak yang sudah ada. Hal ini meningkatkan kecepatan pengembangan, karena masing-masing fungsionalitas tidak harus ditulis dari scratch.

Inovasi

Dengan datangnya aplikasi baru, seluruh industri dapat mengalami perubahan. Bisnis dapat merespons dengan membuat perubahan pada tingkat API tanpa harus menulis ulang seluruh kode.

Ekspansi

API memberikan peluang yang unik bagi bisnis untuk memenuhi kebutuhan kliennya di seluruh platform. Misalnya, API peta memungkinkan integrasi informasi peta melalui situs web, Android, iOS, dll.

Kemudahan Pemeliharaan

API bertindak sebagai gateway antara dua sistem. Masing-masing sistem wajib melakukan perubahan internal agar API tidak terpengaruh. Dengan cara ini, perubahan kode apa pun di kemudian hari yang dilakukan oleh salah satu pihak tidak akan berdampak pada pihak lainnya.

Keamanan REST API

Token Autentikasi

Digunakan untuk mengizinkan pengguna melakukan panggilan API. Token autentikasi memeriksa bahwa pengguna adalah pengguna yang sebenarnya dan memiliki hak akses untuk panggilan API tersebut.

Contoh: JWT (JSON Web Tokens), OAuth

Kunci API

Memverifikasi program atau aplikasi yang membuat panggilan API. Kunci-kunci ini mengidentifikasi aplikasi dan memastikan aplikasi tersebut memiliki hak akses yang diperlukan.

Penggunaan: Untuk API publik yang tidak memerlukan autentikasi pengguna

OAuth

Protokol otorisasi yang memungkinkan aplikasi untuk mengakses sumber daya server atas nama pengguna tanpa harus membagikan kredensial mereka.

Protokol: OAuth 1.0, OAuth 2.0

Langkah-Langkah Membuat API

  1. Perencanaan: Tentukan tujuan, fungsi, dan target pengguna API
  2. Desain: Buat struktur endpoint, metode HTTP, dan format data
  3. Implementasi: Kembangkan API menggunakan bahasa pemrograman dan framework
  4. Pengujian: Uji setiap endpoint untuk memastikan fungsionalitas yang benar
  5. Dokumentasi: Buat dokumentasi yang jelas untuk pengembang
  6. Deployment: Publikasikan API ke server produksi
  7. Pemantauan: Pantau kinerja dan penggunaan API secara berkala
